# Workflow

OP DevFlow is a PRD-driven delivery workflow with Spec as the mandatory contract before PRD review.

## Phases

1. PRD Intake
2. Spec Authoring
3. Spec Review
4. Requirement Trace
5. Technical Solution
6. Solution Review
7. Implementation Tasks
8. Implementation
9. Unit Test Plan
10. Unit Test Report
11. Delivery Review
12. Checkpoint and Observability

## Gate philosophy

A gate does not mean no progress. A gate means unresolved risk must be visible and accepted before progress.

## Observable outcome

The feature folder should explain current state without chat history:

- What PRD items exist.
- Which Spec items cover them.
- Which solution sections implement them.
- Which tasks changed code.
- Which unit tests cover acceptance criteria.
- Which risks remain open.
- What the next action is.
