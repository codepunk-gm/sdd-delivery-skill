# Review Rubric

Use findings first. Order by severity.

## Severity

- P0: cannot continue or release
- P1: high risk, should fix before next phase
- P2: medium risk, should track
- P3: minor issue or polish

## Spec Review

Check:
- PRD item coverage
- clear acceptance criteria
- testability
- edge cases
- non-goals
- contradictions
- missing constraints

## Solution Review

Check:
- repo evidence
- architecture fit
- API compatibility
- data and migration impact
- error handling
- security and permissions
- performance
- rollback
- verification strategy

## Delivery Review

Check:
- Spec satisfaction
- trace matrix completeness
- scoped changes
- meaningful unit tests
- unresolved failures
- regression risk
- checkpoint recovery quality
