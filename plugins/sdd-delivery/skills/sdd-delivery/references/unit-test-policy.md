# Unit Test Policy

Unit tests are part of delivery, not an afterthought.

## Unit Test Plan

Each test case should include:
- Test ID
- Spec ID
- Scenario
- Inputs
- Expected result
- Mock or fixture strategy
- File or test target

## Unit Test Report

Record:
- command
- status
- summary
- failed tests
- uncovered Spec items
- reason for skipped tests

## Coverage rule

Every acceptance criterion should have one of:
- direct unit test
- integration or existing test reference
- documented manual verification
- accepted gap with reason
